# 資格勉強アプリ開発プロジェクト
## コンセプト：「気づいたら合格してたアプリ」
意志ゼロでも開いてしまう・続いてしまう設計。継続設計そのものが差別化軸。

---

## PDCAサイクルのルール
タスクを受けたら必ず以下の流れで考えること：

- **P（Plan）** : アーキテクト → SE が設計・要件定義
- **D（Do）**  : UIデザイナー → プログラマー が実装
- **C（Check）**: デバッガー → QA が検証
- **A（Act）** : マーケター が改善提案 → Pへ戻る

## 利用可能なロール（スキル）
| スキル名 | 役割 |
|---|---|
| pdca-manager | PDCAを自動で回すオーケストレーター |

## 技術スタック
- **フェーズ1**：Webアプリ（HTML / JavaScript）でプロトタイプ
- **フェーズ2**：Android APK化（TWA または React Native）
- **フェーズ3**：Google Play ストア公開
- 広告：Google AdMob（セッション終了後）
- 画像：Stable Diffusion生成の静止画（PNG）
- 問題データ：JSONファイル管理（初期：各資格50問〜）
- 連続日数管理：LocalStorage（Web版）→ネイティブストレージ（アプリ版）
- 開発環境：VS Code + Claude Code（Windows）

## 核心機能：連続ログイン成長システム
**このアプリ最大の差別化ポイント。既存アプリに全くない。**

| 連続日数 | 問題数 | プレイ時間 | キャラの変化 |
|---|---|---|---|
| 1日目 | 3問 | 約2分 | 「初めまして♡ 一緒に頑張ろう」 |
| 2日目 | 4問 | 約3分 | 「また来てくれた！嬉しい」 |
| 7日目 | 7問 | 約6分 | 【限定キャラアンロック】 |
| 30日目 | 10問（上限） | 約8分 | レアシーン |
| 途切れたら | 3問にリセット | 約2分 | 「待ってたよ...また最初から」 |

### 継続の心理設計
- **ツァイガルニック効果**：少し物足りないで終わる → 明日また開く
- **損失回避バイアス**：積み上げたものを失いたくない恐怖
- **間欠強化**：たまにランダムでレアセリフ・ボーナス問題が出現
- **キャラ愛着**：キャラに会いたくて開く。勉強のためではなく。

## 画面構成
| 画面名 | 内容 |
|---|---|
| ホーム画面 | 連続日数🔥・今日の問題数・キャラ・「今日も始める」ボタン |
| 問題画面 | 問題文・選択肢・残り問題数 |
| 解説画面 | 正解 or 不正解・キャラのリアクション・解説テキスト |
| 強制終了画面 | 「今日はここまで！また明日ね♡」・明日の問題数予告・広告 |
| キャラ一覧 | アンロック済み・未アンロックキャラ |
| 設定画面 | 通知時間設定・資格選択 |

---

## キャラクター設計
- キャラ数（初期）：1キャラ（7日で2体目・30日で3体目アンロック）
- 画像：Stable Diffusion生成の水着美女キャラ（PNG静止画）
- セリフ：正解用15・不正解用15・レアセリフ10・終了時5・通知用10
- 親密度：連続日数で「初対面→打ち解け→親友」にトーン変化
